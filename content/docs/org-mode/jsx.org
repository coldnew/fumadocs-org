#+title: JSX Support in Org-mode
#+description: Using JSX directly in Org-mode documents

* JSX Support

Org-mode now supports embedding JSX directly using =#+JSX:= (or =#+jsx:=) directives. Unlike HTML blocks that need conversion, JSX blocks contain React-compatible syntax that can be used directly in MDX.

** Basic JSX Usage

You can embed JSX elements using the =#+JSX:= directive:

#+JSX: <button className="btn btn-primary" type="button">Click me</button>

This JSX will be preserved as-is in the exported MDX output.

** JSX Attributes

JSX attributes work exactly as they do in React:

#+JSX: <div className="card" data-testid="my-card" aria-label="Card component">
  <h3>Card Title</h3>
  <p>Card content</p>
</div>

** Conditional Rendering

You can use conditional expressions in JSX:

#+JSX: <div className="notification">
  {true ? <span className="success">Operation completed</span> : <span className="error">Operation failed</span>}
</div>

** Complex JSX Structures

For more complex JSX, use proper indentation:

#+JSX: <div className="container">
  <header className="header">
    <h1>Welcome</h1>
  </header>
  <main className="content">
    <p>This is the main content area.</p>
  </main>
</div>

** Multi-line JSX Export Blocks

For complex multi-line JSX structures, use =#+begin_export jsx= blocks:

#+begin_export jsx
<div className="dashboard">
  <aside className="sidebar">
    <nav>
      <ul>
        <li><a href="#home">Home</a></li>
        <li><a href="#profile">Profile</a></li>
        <li><a href="#settings">Settings</a></li>
      </ul>
    </nav>
  </aside>
  <main className="main-content">
    <h2>Dashboard Content</h2>
    <div className="stats-grid">
      <div className="stat-card">
        <h3>Total Users</h3>
        <span className="stat-number">1,234</span>
      </div>
      <div className="stat-card">
        <h3>Active Sessions</h3>
        <span className="stat-number">89</span>
      </div>
    </div>
  </main>
</div>
#+end_export

** Using React Components

You can reference React components that will be available in your MDX context:

#+JSX: <div className="custom-wrapper" style={{ padding: '1rem', border: '1px solid #ccc' }}>
  <h4>Custom Component Example</h4>
  <p>This demonstrates JSX structure that could be replaced with actual components.</p>
</div>

** Best Practices

- Use JSX blocks when you need React-specific features like event handlers or state
- Ensure JSX syntax is valid - the converter doesn't validate JSX syntax
- Import any custom components at the top of your MDX files
- Use proper React naming conventions (camelCase for attributes)
- Test your JSX in a React environment to ensure it works correctly

** Comparison with HTML Blocks

| Feature | #+HTML: | #+JSX: | #+begin_export html | #+begin_export jsx |
|---------|---------|--------|---------------------|-------------------|
| Syntax | HTML | JSX | HTML | JSX |
| Conversion | HTML → JSX | Direct output | HTML → JSX | Direct output |
| Event handlers | Converted to JSX | Already JSX | Converted to JSX | Already JSX |
| Attributes | class → className | Already correct | class → className | Already correct |
| Multi-line | No | No | Yes | Yes |
| Use case | Static HTML | Interactive components | Complex static HTML | Complex interactive JSX |

** Generic Export Blocks

For other export formats, use generic =#+begin_export <type>= blocks. The content is exported as-is without any processing:

#+begin_export markdown
# Raw Markdown

This content remains as **markdown** syntax.
#+end_export

#+begin_export latex
\[ E = mc^2 \]
#+end_export

Choose =#+JSX:= when you need React functionality, and =#+HTML:= when you have static HTML that needs conversion.
